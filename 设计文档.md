# Skill 设计文档：思维拆解 skill
> 底层：广义思维方式 · 载体：辩论结构

---

## Skill 定位

**不蒸馏人，不蒸馏书——蒸馏在压力下构建和修正判断的认知操作序列。**

适合：面对复杂问题、需要构建观点、需要预判攻防、需要做艰难取舍的场景。

---

## 核心操作体系（共5层）

| 层级 | 操作名 | 做什么 |
|------|--------|--------|
| 1 | **切入角度** | 同一个命题，从哪个侧面进？为什么选这个而不是那个？ |
| 2 | **立论** | 核心论点是什么？它的前提是什么？前提成立吗？ |
| 3 | **攻防** | 对方会从哪里打你？你最弱的地方在哪？怎么守或主动转化？ |
| 4 | **结变** | 什么时候放弃一个点转向？这个判断的成本是什么？ |
| 5 | **价值收束** | 这件事最终落在什么价值层面？为什么它重要？ |

---

## 两种模式

### 模式 A · 拆解模式（主动给你答案）

> 你扔一个问题/命题，skill 自动过一遍5层操作，输出一份完整拆解报告

**触发方式**：用户输入命题或问题 → 选择"拆解模式" → 输出报告

**输出格式**：
```
【命题】xxx

【切入角度】选择了X维度切入，而非Y，原因是：...
【核心立论】...（含前提说明）
【主要攻防点】对方可能从A/B/C打，守的方式是...
【结变建议】如果遇到X压力，建议在此处转向...
【价值落点】这个问题最终指向的是...
```

---

### 模式 B · 陪练模式（苏格拉底式追问）

> skill 变成思维教练，不给答案，只追问——逼用户自己推导

**触发方式**：用户输入命题或问题 → 选择"陪练模式" → 进入对话

**追问逻辑**：
1. "你选这个角度切入，你排除了哪些其他可能？"
2. "你的核心论点的前提是什么？这个前提你确认成立吗？"
3. "最容易被攻击的地方在哪里？"
4. "如果你必须在A和B之间放弃一个，你放哪个，为什么？"
5. "这件事为什么重要？重要给谁？"

**关键原则**：不给答案，只追问，让用户自己推导出来。遇到用户给出模糊答案时，继续追问具体化。

## 素材来源建议（蒸馏什么）

| 素材类型 | 价值 | 说明 |
|--------|------|------|
| 真实辩题+交锋记录 | ⭐⭐⭐⭐⭐ | 有来回攻防，有取舍轨迹 |
| 有推导痕迹的访谈/长文 | ⭐⭐⭐⭐ | 能看到"为什么这么想"的过程 |
| 真实决策案例（含走错的路）| ⭐⭐⭐⭐ | 错误比正确更能暴露思维结构 |
| 书（只取推导部分，不取结论）| ⭐⭐⭐ | 需要二次加工，提取思维动作 |
| 某人的观点汇总 | ⭐ | 只有结论，无推导轨迹，价值最低 |
