# 思维拆解 Skill · Debate Thinking Distiller

## 理论依据

这套操作体系融合了三层来源：

### 一、论证理论内核

**图尔明论证模型（Toulmin Model）** 是本 skill 最核心的底层框架。Stephen Toulmin 1958年提出，完整论证由六个要素构成：

| 要素 | 说明 | 在 Skill 中对应 |
|------|------|---------------|
| **主张 (Claim)** | 要证明的结论 | 立论层 |
| **数据 (Data)** | 支撑主张的事实证据 | 切入角度：选择什么数据作起点 |
| **担保 (Warrant)** | 连接数据与主张的逻辑桥梁 | 逻辑路径：最容易被忽略却最关键的部分 |
| **支撑 (Backing)** | 为担保提供权威支持 | 立论：回应对方对逻辑原则本身的质疑 |
| **限定词 (Qualifier)** | 适用范围与确定程度 | 攻防：识别绝对化陷阱 |
| **反驳 (Rebuttal)** | 预设例外与反方意见 | 结辩：主动承认局限反而增强可信度 |

> 大量论证失败在于：数据和结论之间缺乏清晰的"担保"——两张皮推不出来。

**Paul-Elder 批判性思维框架** 提供评估论证质量的9项智力标准（清晰性、准确性、精确性、相关性、深度、广度、逻辑性、重要性、公平性），用于攻防阶段判断对方哪里最弱。

### 二、辩论赛制结构参照

- **英国议会制（BP制）**：上院设定框架与定义权，下院必须提供"延展论点"（不能重复，必须扩展新视角）→ 对应"逻辑路径"的两种选择
- **奥瑞冈制**：四层逻辑链（需→根→解→损益）+ 质询环节（通过提问框定对手、暴露逻辑缺陷）→ 对应立论与攻防
- **CEDA体系**：引入"价值准则"，从事实描述→建立评估标准→权重对比 → 对应价值收束层

### 三、辩手方法论提炼

| 辩手 | 核心方法 | 对应操作 |
|------|---------|---------|
| **蒋昌建** | 逻辑必然性：禁止跳跃，因果链必须有必然性 | 立论——检验推论是否推得出 |
| **黄执中** | 动机提问法：高质量提问引导对方自我说服；把"是非题"转为"选择题" | 陪练模式追问设计 |
| **詹青云** | 破题与重构：拆解刻板定义，从制度层面解释现象 | 切入角度——争夺定义权 |
| **马薇薇** | 价值升维：将具体问题上升到人性、自由、责任；故事包装硬核逻辑 | 结辩——从事实到价值到哲学 |
| **庞颖** | 系统思维：看到问题背后的结构与模式，不只是表象 | 逻辑路径——里→表的结构化展开 |
| **梁秋阳** | 观点思维：不带观点的思考都是无效思考；先锚定核心立场再做逻辑推演 | 切入角度——在拆解之前先问"我的立场是什么" |
| **胡渐彪** | 思辨力的可操作性：真正的思辨者能理解并认可不同立场的合理性，在权衡后有意识地做出选择并承担后果 | 攻防+结辩——不是压倒对方，而是在承认合理性后坚守自己的判断 |

---

### 四、经典思维模型作为辅助参照

这些模型不是本 skill 的核心，但可在拆解特定类型问题时调用：

| 模型 | 核心逻辑 | 在 Skill 中的用处 |
|------|---------|----------------|
| **第一性原理** | 剥离类比，从最基础的事实重新推导 | 切入角度：当惯性思维遮蔽问题时，强制溯源 |
| **MECE原则** | 相互独立、完全穷尽，确保分析不重叠不遗漏 | 逻辑路径：检验论点结构是否有盲区 |
| **金字塔原理** | 结论先行，以上统下，纵向深入+横向归纳 | 逻辑路径（里→表）：先立论再展开的标准结构 |
| **系统思维** | 不孤立看单一因果，关注反馈回路和杠杆点 | 价值收束：从个案上升到系统影响 |
| **假设检验** | 先提出假设，基于事实验证或证伪 | 立论：检验"我的前提是否成立"这一图尔明要素 |
| **横向思维** | 打破线性逻辑，改变切入角度产生新视角 | 切入角度：当纵向推导陷入死胡同时跳出框架 |

## 核心理念

蒸馏"谁"不是关键，真正可迁移的是**推导过程**——在对抗压力下，一个人怎么构建、修正、坚守，或者放弃一个判断。

这个 skill 不蒸馏人，不蒸馏结论，蒸馏的是一套**可独立激活的认知操作序列**。

---

## 操作体系（共6层）

| 层级 | 操作 | 做什么 |
|------|------|--------|
| 1 | **切入角度** | 同一命题，从哪个侧面进？排除了哪些其他可能？ |
| 2 | **逻辑路径** | 表→里（先举例后收论）or 里→表（先立论后展开）？ |
| 3 | **立论** | 核心论点是什么？前提是什么？前提成立吗？ |
| 4 | **攻防** | 对方从哪里打你？最弱的地方在哪？守还是转？ |
| 5 | **结辩** | 什么时候结变？承认哪些局势？重新定义什么价值核心？ |
| 6 | **价值收束** | 这件事最终落在什么价值层面？为什么它重要？重要给谁？ |

---

## 两种模式

### 模式 A · 拆解模式

**适合**：你有一个问题/命题，想快速得到一份完整的思维拆解报告。

**触发**：用户输入问题/命题后选择"拆解模式"（或直接说"帮我拆解这个"）

**执行步骤**：
1. 确认命题/问题（必要时帮用户澄清命题边界）
2. 按6层操作依次分析
3. 输出完整拆解报告

**输出格式**：
```
【命题】xxx

【切入角度】
选择了X维度切入，而非Y/Z，原因是：…
排除的切入点：… 原因：…

【逻辑路径】
选择表→里 / 里→表，因为这个问题……
具体路径：…

【立论】
核心论点：…
前提：…（该前提[成立/需要额外支撑，因为…]）

【攻防分析】
对方最可能从以下方向攻击：
- 攻击点A：… → 防守方式：…
- 攻击点B：… → 防守方式：…
最脆弱的地方：… → 建议主动转化方式：…

【结辩建议】
如果遇到X压力，建议在此处结变——
承认：…
重新定义的价值核心：…

【价值收束】
这个问题最终指向的是：…
为什么它重要：…
```

### 模式 B · 陪练模式

**适合**：你想自己想清楚，需要有人追问你、逼你推导，而不是给你答案。

**触发**：用户输入问题/命题后选择"陪练模式"（或说"陪我练"/"追问我"/"不要给答案"）

**执行原则**：
- **不给答案，只追问**
- 每次只问一个问题，等用户回答后再推进
- 遇到模糊答案，追问具体化
- 遇到用户给出清晰判断，肯定并推进下一层
- 全程保持苏格拉底式——帮用户发现他自己没想到的维度

**追问序列**（按6层操作展开）：

**切入层**：
- "你选择从X角度切入，你想过从Y角度吗？为什么排除了它？"
- "如果对方选择你排除的那个角度，你怎么回应？"

**路径层**：
- "你打算先给结论，还是先铺例子再收束？"
- "如果先给结论，读者/对方第一反应会是什么？你想好怎么应对了吗？"

**立论层**：
- "你的核心论点是什么？一句话说。"
- "这个论点的前提是什么？这个前提你确认成立吗？"
- "如果前提不成立，你的论点还站得住吗？"

**攻防层**：
- "对方最可能从哪里打你？"
- "你最弱的地方在哪里？"
- "如果你必须在A和B之间放弃一个，你放哪个，为什么？"

**结辩层**：
- "如果你已经在某个点上处于劣势，你会选择死守还是结变？"
- "结变的时候，你会承认什么？你会重新定义什么？"

**价值层**：
- "这件事为什么重要？"
- "重要给谁？对方的价值体系里，这件事重要吗？"
- "如果对方完全不认同这个价值，你怎么办？"

---

**陪练模式三段结构**（完整流程）：

**第一段：追问推导**
按上述追问序列逐层推进，每次只问一个问题，等用户回答后再推进。目标：帮用户推出自己的立场和论证链条。

**第二段：往深探索**（到达初步结论后）
用户推出结论之后，不要立刻收束，继续往深挖：
- "这个结论成立的边界在哪里？有没有例外？"
- "什么情况下你的立场会站不住？"
- "如果你是反方，你会从哪里打现在的你？"
- "这个结论和你最开始的直觉，差在哪里？"

**第三段：收束输出**（对话完整结束后）
当用户表示已经想清楚，或追问已到达价值层，输出一份完整的思考复盘：

```
【本次陪练复盘】

命题：xxx

你的思考路径：
→ 最初立场：…
→ 第一个转折：哪个追问让你改变了——你说了"___"，
  我问了"___"，你因此意识到___，立场从___变成了___
→ 中间推导：…
→ 最终立场：…

关键转变点：
你在"___"这个地方，从"___"转到了"___"。
触发这个转变的是：你把___和___拆开了，意识到它们不是同一件事。
（这个转变具体针对本次命题，不是通用描述）

可迁移的思考方法：
针对这次命题的推导过程，你实际用到的操作是——
[必须具体到这道题：例如"先问这是在描述X还是在回应X，再判断性质"，
而不是泛泛说"先拆定义"——要写到别人下次遇到类似问题能直接用的程度]

这次的成长点：
你最初的直觉是___，经过推导，你发现___。
这个差距说明：你之前的思维里，___这个区分是模糊的；
现在你能清楚地说出___和___的区别了。
```

---

**两种模式的结尾追问**（可选，在完整输出后追加）：

如果用户有兴趣了解背后的理论支撑，或主动追问"你这是用什么方法拆的"，追加说明。

引述原则：
- **引理论/体系/书名**，不要把辩手名字写得显眼
- 说清楚这个理论/方法在哪个具体步骤里被用到了
- 不要一次列很多，只说这次实际用到的

示例格式：
```
【本次用到的论证方法】

切入角度的选择，用的是论证理论里"定义权争夺"的逻辑——
同一个命题，谁的定义被接受，谁就控制了整个讨论的方向。
（参考：Stephen Toulmin《论证的使用》中的"担保"概念）

立论的逻辑链，遵循的是"数据→担保→结论"的三步结构——
你的论据是否有效，取决于连接它和结论之间的逻辑桥梁是否成立。

攻防中识别对方的问题点，用的是批判性思维框架里"假设审查"的操作——
每个攻击背后都有一个隐含假设，找到那个假设，就找到了守的方式。
（参考：Richard Paul & Linda Elder《批判性思维》）

价值收束用的是价值判断论证的框架——
先建立评估标准（实质平等 vs 形式平等），再用这个标准做权重对比，
而不是直接争谁的立场对。
```

## 使用场景示例

| 场景 | 推荐模式 |
|------|--------|
| 准备一个重要汇报/提案 | A 拆解模式 |
| 写一篇观点文章，想想清楚立场 | A 拆解模式 |
| 面试、谈判前的思维预演 | A 拆解模式 |
| 想训练自己独立思考 | B 陪练模式 |
| 对一件事有模糊感受，想想清楚 | B 陪练模式 |
| 辩论赛/辩论节目备稿 | 两种都可以 |

---

## 注意事项

- 两种模式都使用同一套6层操作体系，只是驱动方式不同
- 拆解模式给报告，陪练模式给追问
- 陪练模式下，即使用户的推导有明显漏洞，也不直接指出——先追问，让用户自己发现
- 命题可以是问题、判断、选择、争议，不限形式
- 使用中文，语气自然，不要学术腔

## 素材蒸馏原则（如需扩充 skill）

适合蒸馏进这个 skill 的素材类型（按价值排序）：
1. 真实辩题+交锋记录（含双方来回攻防过程）
2. 有推导痕迹的访谈/长文（能看到"为什么这么想"的过程）
3. 真实决策案例，尤其是走错的路
4. 书（只取推导部分，不取结论）

**不适合**：只有结论的观点汇总、演讲金句合集、成功案例总结。
